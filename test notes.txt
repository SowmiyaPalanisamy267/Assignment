Feature:Reset password via OTP
User story:
As an existing user
i want to reset my password via OTP
so that i can enter a new password

Acceptance criteria:
1.Entered email id already should be registered
2.Entering wrong email id should displays error
3.Entering blank  should display validation message
4.After entering correct OTP received in email, user should be allowed to reset password

Test scenarios
1.OTP Generation: Verify that providing a registered email address triggers a successful OTP delivery to that inbox within the specified timeframe.

2.Password Update: Verify that after entering a valid new password and confirmation password, the system updates the password and allows the user to log in with the new credentials.
3.User Notification: Ensure the user receives a confirmation email or on-screen message after a successful password change.
4.Invalid OTP Entry: Verify that entering an incorrect, alphanumeric, or empty OTP displays a clear "Invalid OTP" error message.
5.Expired OTP: Attempt to use an OTP after its validity period (e.g., after 5–10 minutes) and confirm the system rejects it.
6.Unregistered Email: Enter an email address not in the database and verify the system does not send an OTP (ensure the message is consistent with valid emails to prevent enumeration).
7.OTP Reuse: Confirm that an OTP becomes invalid immediately after it has been used once successfully.
8.If a user makes multiple reset requests, ensure only the most recent OTP remains active and all previous ones are invalidated.
